# 待办 backlog

## [P1] 超大 PDF 自动分片 + 再合并（全程无需人工写 YAML / 断点）

### 背景

- 当前 `make` 只处理 **单个** `input/<STEM>.pdf`；Docling/显存/时间在 **千页级** PDF 上容易踩坑，历史上曾通过**手工**按章节或页段拆分、分多轮跑、再拼 MD 解决。
- 上游大仓曾有 `split_pdf_from_yaml.py` 等 **按 YAML 计划** 拆 PDF 的工具，依赖人**事先**定好每段起止页，**不满足**「自动、免鉴别」目标。

### 目标（验收口径）

1. **触发**：当输入 PDF 超过可配置阈值（例如 **页数** 与/或 **文件大小**）时，管线**自动**进入分片流程，无需用户判断「要不要拆」。
2. **分片策略**（实现时二选一或组合，写进配置/环境变量，有合理默认）：
   - 按固定 **页块**（如每 80～150 页一段，可随硬件调参）；
   - 或先探测页数/资源，**自适应**块大小（失败则重试小块）。
3. **执行**：对每段调用与今日 `a1-convert` 等价的转换，产物写入**临时或分片输出目录**（如 `output/_parts/<STEM>_part001.md`…），**日志连贯**、可 `tail`。
4. **合并**：自动拼接为**一份**与当前约定一致的 `output/<STEM>.md`（及必要的 `images.json` 等侧车文件的合并/路径重写规则需单独定义，避免图引用断裂）。
5. **后续步骤**：`a2` 及以后仍对 **合并后** 的单文件跑，**不改变** `make a-all` / `make all` 的对外语义；即用户仍 `make all`，**内部**在 `a1` 前/内完成分片与合并。

### 非目标（初版可不做）

- 智能按「章标题」切分（需布局语义，复杂度高；可与固定页块分阶段做）。
- 自动修复合并处表格跨页断行（可后续增强）。

### 实现时可复用/参考

- 系统依赖：`qpdf` 或 `pypdf` 做纯页范围切分（与 `ai-doctool2-master/split_pdf_from_yaml.py` 思路类似，但**参数由程序生成**，不从 YAML 手填）。
- `Makefile`：新增如 `a0-parts` 或把逻辑放进 `a1-convert` 的包装脚本，避免破坏现有 `STEM` 单文件入口。
