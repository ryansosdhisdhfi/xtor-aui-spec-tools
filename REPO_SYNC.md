# `py/` 脚本与上游工具链

## 本仓库里的 `py/`

本目录包含管线所需 **Python 脚本**，与 **Aidoc 系工具链**（`aidoc_*`、图链、RAG 等）为**同一套能力**的随包副本，便于单仓 clone 即可运行。

## 何时需要「对齐」上游

若上游工具链某脚本有**行为更新或缺陷修复**，可将对应文件从**你们维护的 Aidoc 工具链根目录**合并到本仓 `py/`，再跑一次冒烟（如 `make check` 后 `a1-convert`）。

不必在每次无关变更后机械同步；以**行为是否分叉**为准。

## 不要删 `py/`

`Makefile` 使用 `$(PY)/脚本名`；删除 `py/` 会导致本仓无法按现设计运行，除非改为改为只引用 `$(REPO)` 下的脚本（需改 Makefile，由团队决策）。
